The Internet

HTTP

HTTP and DNS handle the sending and receiving of web files. HTTP is a way that computers communicate with each other. When you open a web browser and you type in the web address, it opens the website you want to visit. When you type in a web address, the computer talks to a server and asks it for a website, and they communicate in HTTP or hypertext transfer protocol. If a user is trying to login to a website, the computer sends a message to the server by the way of a “get” command. Get commands retrieve information from the server to the computer.

Information on the internet is transferred in terms of bits, which is simply an on or off signals. Bits are grouped together in groups of 8 bits, which makes one byte, and 1000 of these bytes make a kilobyte. These can be considered the atoms of the information, or the most basic building blocks of the information being sent around. The latency of the system is the measure of time it takes for one bit to travel from one node to another. Using copper cable, such an ethernet cable, one can easily and cheaply transmit data, but after 80 meters, signal loss becomes too much to handle. Fiber optic cables use light pulses to increase the bandwidth immensely, but they are expensive and fragile, and require a lot of supporting infrastructure. Finally, radio systems such as WiFi are cheap and easy to implement, but only provide a good signal within a range proportional to their power.

IP Addresses

What are they?

Vint Cert and Bob Kahn invented the Internetworking Protocol, which laid the ground work for todays modern internet. Given that the internet is just a mesh of many different networks, how does one communicate from one node to another with minimal interruption? The internet in the words of its creator Vint is that the Internet is “ a design philosophy and an architecture expressed in a set of protocols” The physical technologies that want to join just need to follow the protocol. This type of IP address is called IPv4, with capacity for 4 billion unique addresses. A newer type of IP address called IPv6 is being implemented to allow more devices with no congestion.

DNS: What is it?

When you request a website, you type its domain name into your web browser. This domain is registered with ICANN, and it hosted on a domain name server. Often times, the domain name server that you connect to will not know what the IP address of the domain you are trying to connect to is. It will then reach out to other servers to try and find the correct IP address, and once it does, it will return that information, as well as keep a log of that information in its cache in case you or someone else needs to access that domain again in the near future.
